HogMatcher

This class represents an image template matcher based on the "histogram of oriented gradients" algorithm.

Methods

__init__() This method creates a new HogMatcher.
match() This method matches the given template image within the given image.

Remarks

  • The histogram of oriented gradients (HOG) is a feature descriptor used in computer vision and image processing for the purpose of object detection. The technique counts occurrences of gradient orientation in localized portions of an image. The AutoScript product provides access to this algorithm through the HogMatcher class.
  • The algorithm is particularly effective for sub-pixel localization of fiducial marks in SEM/FIB images, which often feature "X" or "+" shapes, showing gradients in predominant angles.
  • The HOG algorithm is implemented on the server side of AutoScript, so images must be transferred from the client to the server, which introduces some overhead.

Examples


        import autoscript_toolkit.vision as vision_toolkit
        import autoscript_toolkit.user_interface as ui_toolkit
        from autoscript_toolkit.template_matchers import *
    
        # Acquire one full-frame image in the active view
        settings = GrabFrameSettings(resolution="1536x1024", dwell_time=1e-6)
        image = microscope.imaging.grab_frame(settings)
        
        # Acquire one reduced area image to serve as a template
        settings.reduced_area = Rectangle(0.05, 0.05, 0.10, 0.10)
        template = microscope.imaging.grab_frame(settings)
        
        # Locate the template in the original image
        hog_matcher = HogMatcher(microscope)
        location = vision_toolkit.locate_feature(image, template, hog_matcher)
        ui_toolkit.plot_match(image, template, location.center_in_pixels)
