Template matchers

This module offers various implementations of image template matchers - classes designed to accurately locate a template image within a larger image.

These matchers are used throughout the AutoScript toolkit for practical purposes. For example, the drift correction module uses these classes to measure sample drift during extended patterning tasks. Additionally, the locate_feature() function in the vision module provides an easy way to call the matching algorithm directly in client code for image processing tasks.

The module is designed for extendibility. If the provided matcher implementations do not meet your specific requirements, you can create your own pattern matcher class by extending the TemplateMatcher base class and integrate it with the existing drift correction or vision toolkit functions.

Classes

CognexMatcher This class represents an image template matcher based on the Cognex PatMax algorithm.
HogMatcher This class represents an image template matcher based on the "histogram of oriented gradients" algorithm.
OpencvTemplateMatcher This class represents an image template matcher based on the OpenCV template matching algorithm.
TemplateMatcher This is a base class for all image template matchers compliant with the AutoScript toolkit modules.

Enumerations

OpencvComparisonMethod This enumeration lists comparison methods to be used by the OpenCV template matching algorithm.
