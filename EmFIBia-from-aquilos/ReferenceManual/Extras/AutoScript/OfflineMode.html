Offline mode

In offline mode, the AutoScript server operates disconnected from the microscope and forwards all API calls to the built-in SEM simulator.

AutoScript in offline mode can be installed on any computer that meets the minimum hardware requirements specified in the installation manual. Offline mode supports both local and remote scripting configurations, although the local configuration is recommended.

Simplistic modeling

The scanning electron microscope is a complex device. The integrated microscope model does not attempt to accurately simulate all of this complexity. Most API functions intended to control the state of the microscope function as a simple read/write cache. A value written by the property setter is stored and returned when the getter is called. This means, for example, that changing the value of beam shift has no effect on other optical properties.

The state of the simulator is not persisted. When AutoScript server is restarted, the simulator resets to its initial state and the current state is lost.

Image acquisition

Images output from the detectors or cameras are not visually affected by the state of the microscope or acquisition settings. Static images are provided, but in the correct size and format. AutoScript allows you to upload a custom simulation image using the imaging.set_image() function. This can be particularly useful for simulating parts of the workflow that involve image processing.

Image metadata simulation is limited. Images are created with static metadata structures that do not reflect the simulated system state. 

Auto functions

Offline auto functions are not simulated, except for simple input parameter checks.

Offline system properties

  • Dual beam system, i.e. system that has both electron and ion beams.
  • System name is "Offline", its platform software version corresponds to the AutoScript server version.
  • Microscope serial number is "Offline-001".
  • Available detectors: ['BSD', 'CDEM', 'CRD', 'DualBSD', 'ETD', 'External', 'GAD', 'GBSD', 'GSED', 'HiResOptical', 'HiResOpticalLowMag', 'ICE', 'InColumnBSD', 'IR', 'IR2', 'IRCameraDetector', 'LFD', 'LVD', 'LVSED', 'Mix', 'PMT', 'QuadBSD', 'SED', 'SingleBSD', 'TLD', 'TLD2'].
  • Available GIS ports: ['Pt dep', 'W dep', 'Al dep', 'Au dep', 'C dep', 'W'] and MultiChem.
  • Available electron beam optical modes: ['FieldFree', 'Immersion'].
  • 5-axes bulk stage and 5-axes retractable compustage.
  • Manipulator (EasyLift) in EX version.
  • Heating stage with temperature range 10..1000 °C.
  • Real time monitor is not simulated, calls to RTM functions return exceptions.
  • Cap probe is not simulated, specimen.cap_probe.is_installed returns False.
  • LoadLock is not simulated, specimen.loadlock.is_installed returns False.
  • Sample cleaning device is not simulated, specimen.sample_cleaning.is_installed returns False.

System initial state

  • View 1 and 3 are configured with electron beam, View 2 and 4 with ion beam.
  • Chamber is pumped to high vacuum.
  • All GIS needles retracted, MultiChem retracted.
  • All retractable detectors retracted.
  • Scanning conditions of both beams: full frame mode, resolution 768x512, dwell time 1 µs.
  • Electron beam optical mode: FieldFree.
  • No patterns in any of the 4 views.
  • Horizontal field width: 10 µm.
  • Stage homed, at position x=0, y=0, z=0, r=0, t=0, Z axis unlinked.
  • Compustage homed, at position x=0, y=0, z=0, a=0, b=0, unlinked.
