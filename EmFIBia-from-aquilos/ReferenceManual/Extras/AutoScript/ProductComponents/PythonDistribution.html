Enthought Deployment Manager (EDM)

When the AutoScript product is installed, an initial Python environment with the name "AutoScript" is created. You can manage packages within this environment as well as create or remove new environments using the standard tooling provided by Enthought - the Enthought Deployment Manager command line tool - "EDM". For more information about using the EDM tool, see the Enthought website online at https://docs.enthought.com/edm.

Please note that AutoScript installs its own version of EDM without registering it in the system path. This approach is intentionally taken to prevent interference with other EDM installations. To ensure smooth functionality, we recommend starting the Windows command prompt using the AutoScript EDM Shell shortcut available in the Start menu. By doing so, the command prompt will open with all the necessary environment variables preconfigured to work with the EDM provided by AutoScript.

The Python package management section of the manual provides further details on how to install packages to environments. Additionally, it showcases the usage of the AutoScript Package Manager GUI extension, which allows you to work with packages without relying on the EDM command-line interface.
