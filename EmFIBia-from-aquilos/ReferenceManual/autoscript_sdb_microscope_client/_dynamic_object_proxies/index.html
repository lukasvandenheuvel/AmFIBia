Dynamic objects

Unlike static-tree objects, which have the same lifetime as the root SdbMicroscopeClient class, dynamic objects are created and destroyed during code execution.
A dynamic object can represent a microscope device (for example Multichem) or an object created on-demand, like RectanglePattern for setting up the patterning process.

Dynamic objects are rarely created directly by the client code. Typically, you would use a function on the static tree to create the dynamic object for you. A key characteristic of dynamic objects is that they remain "connected" to the microscope throughout their lifetime. As a result, any action performed on these objects will have an immediate effect on the microscope.
