EdsStream.run_phase_analysis

Performs an analysis of the EDS stream to identify distinct chemical phases within the sample. Also prepares the phase mapping engine, allowing it to generate phase maps upon subsequent calls to get_phase_map(). The stream must be closed before calling this function.

Syntax

run_phase_analysis()
run_phase_analysis(settings)

Parameters

settings EdsPhaseAnalysisSettings Parameters of the phases analysis, primarily specifying properties of the mapping engine.

Return value

EdsPhaseAnalysisResult The analysis result, including the number of identified phases and statistical data for each phase.

Remarks

  • If the settings parameter doesn't specify otherwise, the function will configure the phase mapping engine with default segmentation resolution and filter, namely dynamic object-based segmentation, mean filter and auto strength.
  • If you don't provide the component_count parameter in the settings to advise the analysis on the number of components to search for in the stream, the count will be determined automatically and included in the analysis result object.
  • Running the analysis is a mandatory step before calling the get_phase_map() or get_phase_spectrum() functions.
  • Only one phase mapping engine can be initialized at a time. If you need to generate maps with different engine settings, execute the analyses successively. For instance, first, perform the analysis with the mean filter and generate the initial set of maps, then repeat the process with the median filter to produce a second set of maps.
