SdbMicroscopeClient.connect

Connects the AutoScript client to the AutoScript server.

Syntax

connect()
connect(host, port)

Remarks

Repeated connections are not allowed. To connect to another host, the disconnect() function must be called first.

Examples

The following code snippet shows how SdbMicroscopeClient is initialized and connected to the AutoScript server.

When calling the connection method without parameters, the default microscope computer address 192.168.0.1 is used.


        # Create an AutoScript client object and connect it to the server at the default microscope computer address 192.168.0.1
        microscope = SdbMicroscopeClient()
        microscope.connect()

A custom IP address can be specified to the connection method:


        # Connect to the server at IP address 10.140.15.1
        microscope.connect("10.140.15.1")

Custom hostnames are also supported:


        # Connect to a server with the hostname "HELIOS-NP247"
        microscope.connect("HELIOS-NP247")

If the server is configured to run on a non-standard TCP/IP port, the port number must be specified:


        # Connect to a server at the IP address 10.140.15.1 and port number 6500
        microscope.connect("10.140.15.1", 6500)

The connect() method also accepts arrays of host addresses. The hosts are probed for the presence of the AutoScript server in the order in which they appear in the array. 


        # Connect to a server address of 10.140.15.1. If the connection fails, try localhost and 192.168.0.1 consecutively
        microscope.connect(["10.140.15.1", "localhost", "192.168.0.1"])

For each IP address in an array, a separate port number can be specified, the length of both arrays must be equal:


        # Establish a connection to a server at 10.140.15.1:7500. If the connection fails, try localhost:4000
        microscope.connect(["10.140.15.1", "localhost"], [7500, 4000])
