GetImageSettings.timeout

The maximum wait time, in seconds, for the completion of one frame. Applicable only when wait_for_frame is set to True.

Data type

[Optional] float

Remarks

  • For standard imaging tasks, the timeout value is automatically derived from the image frame time and does not need to be manually set. However, it becomes necessary when additional delays are present, such as during intermittent simultaneous patterning and imaging (iSPI) processes. In these cases, imaging is configured to capture SEM images at specified intervals that do not correspond to the actual frame time, and this interval needs to be reflected by the imaging.get_image() function.
  • If the timeout expires, the imaging.get_image() function returns the partially updated image from the pipeline right away. No exception is thrown.
