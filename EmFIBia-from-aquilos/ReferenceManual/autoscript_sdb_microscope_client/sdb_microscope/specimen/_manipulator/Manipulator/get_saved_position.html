specimen.manipulator.get_saved_position

Returns a saved (named) manipulator position. There are four default named positions. The user can save more using the main user interface, but these positions are inaccessible to AutoScript.

Syntax

get_saved_position(position_name)
get_saved_position(position_name, coordinate_system)

Parameters

position_name str The name of the saved position. You can use values from the ManipulatorSavedPosition enumeration.
coordinate_system str The manipulator coordinate system in which to retrieve the saved position. You can provide a value from the ManipulatorCoordinateSystem enumeration. If not provided, the function will use the default coordinate system.

Return value

ManipulatorPosition The saved position in form of a ManipulatorPosition structure.

Remarks

Care must be taken when reading the saved position in Raw coordinate system. In this system the Z coordinate takes into account the insertion state of the manipulator. Therefore, a saved position returned by this function when the manipulator is inserted is different from the same call made when the manipulator is retracted.
If you want to insert the manipulator into a saved position specified in Raw coordinate system, read its value while the manipulator is retracted.
In the Stage coordinate system, the position does not depend on insertion state, so you can read the saved positions at any time.
