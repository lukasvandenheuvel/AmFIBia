analysis.eds.mapping.reset_stream

Creates a new empty EDS stream object in the microscope platform software and sets it as active. EDS data generated by the subsequent beam activity, such as the number of characteristic X-rays and their positions, are collected in the newly created stream.

Syntax

reset_stream()

Remarks

  • Typically, reset_stream() is called at the start of an EDS mapping experiment to clear any previous cached EDS data resulting from prior beam scanning activity.
  • All existing stream objects referenced in the script before calling reset_stream() remain valid and can be still used for data processing.
  • This function is internally identical to the analysis.eds.reset_spectrum() function. The inclusion of both functions is for contextual clarity.
  • Note that the EDS buffer is owned by the platform software and is shared among all running applications. Effects of resetting the stream are observable, particularly in the main user interface application, where all currently displayed color maps will disappear.

Examples


        # Assuming the active view is configured with the electron beam, scanning stopped
        
        # Create a new empty EDS stream and activate it 
        microscope.analysis.eds.mapping.reset_stream()
        
        # Scan the entire field of view for 10 seconds; the EDS stream will register X-rays generated by the scanning
        microscope.beams.electron_beam.scanning.mode.set_full_frame()
        microscope.imaging.start_acquisition()
        time.sleep(10)
        microscope.imaging.stop_acquisition()
        
        # Get the active stream and close it to enable data processing
        eds_stream = microscope.analysis.eds.mapping.get_stream()
        eds_stream.close()
        
        # Save all data from the stream to disk in PAD format 
        eds_stream.save("experiment.pad")
