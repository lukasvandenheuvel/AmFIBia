analysis.eds.mapping.get_stream

Returns an object that represents the currently active stream of data from the EDS detector. The data includes both the count of characteristic X-rays and positions on the sample from where the X-rays were generated.

Syntax

get_stream()

Return value

EdsStream The EdsStream dynamic object, which provides functions for post-processing acquired data, e.g. generation of map images.

Remarks

The EDS stream data container is owned by the microscope platform software. All data resides on the server side (i.e., on the microscope computer). Obtaining the stream object by calling get_stream(), does not entail copying potentially large data from the server to the client. The returned object serves as a handle for the server-side data.

Examples


        # Assuming the active view is configured with the electron beam, scanning stopped
        
        # Create a new empty EDS stream and activate it 
        microscope.analysis.eds.mapping.reset_stream()
        
        # Scan the entire field of view for 10 seconds; the EDS stream will register X-rays generated by the scanning
        microscope.beams.electron_beam.scanning.mode.set_full_frame()
        microscope.imaging.start_acquisition()
        time.sleep(10)
        microscope.imaging.stop_acquisition()
        
        # Get the active stream and close it to enable data processing
        eds_stream = microscope.analysis.eds.mapping.get_stream()
        eds_stream.close()
        
        # Save all data from the stream to disk in PAD format 
        eds_stream.save("experiment.pad")
