imaging.grab_frame_to_disk

Scans a completely new frame in the active view, saves the image to disk and provides the preview image.

Syntax

grab_frame_to_disk(file_path)
grab_frame_to_disk(file_path, file_format)
grab_frame_to_disk(file_path, file_format, settings)

Parameters

file_path str The path where the image should be saved. The extension specified in this parameter does not affect the file format.
file_format str The format of the file. ImageFileFormat enumeration can be used to select the format.
settings GrabFrameSettings The custom scan settings to be used instead of default values.

Return value

AdornedImage A preview of a microscope image that corresponds to the acquired frame.

Remarks

  • The function initiates the scanning of a new frame and waits until it is completed. The resulting image is saved at the specified location on disk.
  • Custom scan settings can be provided by the settings parameter. If not specified, the actual scan settings are determined as follows:
    • resolution, dwell_time and reduced_area will default to the current settings in the active view.
    • Other settings will fallback to predefined constant default values:
      • bit_depth will be 8.
      • line_integration will be 1.
      • scan_interlacing will be 1.
      • frame_integration will be 1.
      • drift_correction will be False.
  • Maximum resolution accepted in settings is 65536x65536.
  • It is recommended to use one of the standard resolutions for common imaging tasks, that is a resolution from the ScanningResolution enumeration. If a non-standard resolution is specified in settings, the function will use a less effective method of imaging and some of the scan parameters will not be applicable. See User guide: Imaging and User guide: Large image acquisition for more information.
  • If the preview resolution is not specified in settings, it is automatically calculated by an algorithm described in the "Examples" section on this page (function get_default_preview_resolution). The preview image has the same bit depth as the original image.
  • The views parameter, if specified in settings, is ignored by this function.
  • If no file format is specified, TIFF is used.
  • Saving images in non-standard resolutions in the TIFF format causes additional overhead with converting the image file from the native RAW format to TIFF.

Examples


        # Take one image in active view with resolution 4096x3556 by integrating two consecutive frames, save the result to a disk file
        settings = GrabFrameSettings(ScanningResolution.PRESET_4096X3536, dwell_time=1e-6, frame_integration=2)
        microscope.imaging.grab_frame_to_disk("C:\\image.tif", ImageFileFormat.TIFF, settings)
    

        # Take a 16k image, integrate each line twice to reduce noise, save the result to a disk file in the RAW format
        settings = GrabFrameSettings("16384x16384", bit_depth=16, line_integration=2)
        microscope.imaging.grab_frame_to_disk("C:\\image.raw", ImageFileFormat.RAW, settings)
    

        from matplotlib import pyplot as plot
        
        # Take an 8k image, save it to disk and display its preview on a pop-up window
        settings = GrabFrameSettings("8192x8192", dwell_time=50e-9, preview_resolution="512x512")
        preview = microscope.imaging.grab_frame_to_disk("C:\\image.tif", ImageFileFormat.TIFF, settings)
        
        plot.imshow(preview.data, cmap='gray')
        plot.show()
    

        def get_default_preview_resolution(width, height) -> (int, int):
            if max(width, height) <= 6144:
                return width, height
        
            return (width * 768 // height, 768) if width < height else (768, height * 768 // width)
