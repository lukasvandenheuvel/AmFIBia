imaging.set_image

Uploads an image into the active view, including its metadata.

Syntax

set_image(image)

Parameters

image AdornedImage An image in form of AdornedImage instance.

Remarks

  • The image must contain a valid metadata block to identify important microscope parameters and set them to the imaging view.
  • The active scanning device in this view will be switched to the device with which the image was acquired. Attempting to set an image from a non-existent imaging device (such as an ion beam image on a SEM) will result in an error.
  • There is a built-in limitation on the image format that can be provided. Only 8-bit and 16-bit grayscale images are allowed. Also, only images originating from electron or ion devices are allowed.
  • If the set image is in one of the supported standard resolutions and contains a databar section, the image is resized to exclude the databar.
  • The image set with this function is automatically displayed in the main user interface of the microscope ("Microscope Control").
  • In offline mode, the set_image() function can be used to assign a simulation image source to an imaging device.
    Subsequent calls to get_image(), grab_frame() or grab_frame_to_disk() will return images based on the given image source, possibly resized to the target resolution. This mechanism works only for standard scanning resolutions - those from the ScanningResolution enumeration.

Examples


    ion_image = AdornedImage.load('IonImageWithDatabar3072x2188x8bit.tif')
    microscope.imaging.set_active_view(1)
    microscope.imaging.set_image(ion_image)
    
    # The image was loaded into view 1 and resized to 3072x2048 pixels by cutting off the databar part.
    # Active imaging device in view 1 has been switched to Ions.
    # The image is visible in the upper-left display of the Microscope Control application.
    # Subsequently calling get_image() will return the loaded image including metadata.
    # Subsequently calling grab_image() will replace the image with newly scanned one.
