Static tree

The static tree is a fixed hierarchy of objects created when the top-level API class, SdbMicroscopeClient, is instantiated. These objects are organized into logical groups to make API navigation easier. Some objects in the tree act solely as containers for other objects, while the "leaf" objects offer functions for controlling the microscope.

Classes

analysis Provides functions for chemical microanalysis of the sample.
auto_functions Enables execution of the microscope auto functions.
beams Enables control and provides the status of the microscope beams.
detector Enables control and provides the status of the active detector, i.e. the detector currently set in the active view.
gas Enables control of the gas injection system (GIS).
imaging Enables control of the microscope imaging.
patterning Enables control and provides the status of the microscope patterning engine.
service Provides service information.
specimen Enables control and provides the status of sample manipulation devices.
state Provides status of various microscope subsystems.
vacuum Enables control and provides the status of the specimen chamber vacuum.
